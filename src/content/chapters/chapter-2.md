### <font color="#ffc000">2-2. 프롬프트 엔지니어링</font>
- 생성형 AI에서 최상의 결과를 얻기 위해 효과적인 입력값을 설계하는 과정

>[!warning] 나쁜 프롬프트
>```markdown
>거래처에 보낼 이메일을 작성해줘.
>```

>[!important] 좋은 프롬프트
>```markdown
>[Framing] 당신은 5년차 영업 담당자입니다.
>
>[Request] 신제품 출시 안내 이메일을 작성해주세요.
>
>[Reference]
>- 제품명: 스마트 온도조절기 TH-2000 
>- 주요 기능: AI 학습 기반 자동 온도 조절, 30% 에너지 절감 
>- 출시일: 2025년 12월 15일 
>- 특별 할인: 선주문 시 20% 할인
>
>[Format]
>- 존댓말, 격식있는 비즈니스 톤 
>- 제목, 인사말, 본문, 행동유도(CTA), 마무리 인사 포함
>- 300자 이내
>```

### <font color="#ffc000">2-3. 프롬프트 엔지니어링 요소</font>
- Format - 결과물의 형태를 구체적으로 지정하는 요소
	- 텍스트의 형식, 구성, 길이, 말투 등을 포함
	- 예시) 500자 이내의 블로그 포스팅 형식으로 작성해주세요.

>[!quote] Format 변경 연습
>```markdown
>인공지능의 장단점을 설명해주세요.
>```
>```markdown
>1단계: 불릿 포인트 5개로 작성
>2단계: 초등학생용 동화 형식으로 작성
>3단계: 찬반 토론문 형식으로 작성
>4단계: 인포그래픽용 짧은 문구로 작성
>```

- Reference - 결과물 생성 시 참조할 수 있는 자료나 정보
	- 이전 대화 내용을 참조하여 답변 작성
	- 외부 웹사이트나 문서를 첨부하여 정보 인용

>[!example] Reference 활용 연습
>```markdown
>건강한 아침 식사에 대한 블로그 글을 작성해줘.
>```
>```markdown
>건강한 아침 식사에 대한 블로그 글을 작성해줘.
>
>[Reference 정보]
>- 타겟 독자: 아침을 자주 거르는 직장인
>- 참고 데이터: 아침 결식률 30% (2025년 통계) 
>- 강조 포인트: 시간 절약 + 영양 균형
>- 참고 레시피: 오버나이트 오트밀, 그릭 요거트 볼
>```
>```markdown
>아래 기사를 참고하여 건강한 아침 식사에 대한 블로그 글을 작성해줘.
>가시의 핵심 내용을 유지하되, 20대 독자가 읽기 쉬운 캐주얼한 톤으로 재구성해줘.
>
>[Reference 정보]
>https://www.healthlifeherald.com/news/articleView.html?idxno=1051
>```

- Request - 원하는 결과물을 명확하게 전달하는 요소
	- 구체적이고 정확한 요청을 통해 원하는 결과물 획득

>[!note] Request 구체화 연습
>```markdown
>운동 계획을 세워줘.
>```
>```markdown
>다이어트 운동 계획을 세워줘.
>```
>```markdown
>30대 남성, 체중 85kg, 목표 75kg, 주 3회 헬스장 이용 가능, 
>12주 동안 진행할 근력운동과 유산소 운동 계획을 주차별로 상세하게 작성해주세요
>```

- Framing - 사용자가 해결하려는 문제나 상황에 대한 설명
	- AI가 문제나 상황을 이해하는 데 필요한 맥락 제공
	- 예시) 당신은 지금부터 나에게 조언을 해주는 심리상담가입니다.

>[!check] Framing 효과 비교
>```markdown
>파이썬 공부 방법을 알려줘.
>```
>```markdown
>당신은 프로그래밍을 한 번도 해본 적 없는 완전 초보자를 가르치는 강사입니다. 
>파이썬 공부 방법을 알려주세요.
>```
>```markdown
>당신은 자바 개발 경력 5년차 개발자에게 조언하는 시니어 개발자입니다.
>파이썬으로 전환하려는 개발자를 위한 효율적인 학습 방법을 알려주세요.
>```

### <font color="#ffc000">2-4. 프롬프트 엔지니어링 실습</font>

>[!quote] 블로그 주제 프롬프트
>```markdown
>당신은 한국에서 AI 저널리스트로 활동하고 있습니다.
>현재 한국에서 사람들이 가장 관심을 가지는 AI 관련 이슈 20가지를 최신 데이터를 기반으로 작성하세요.
>
>## Instruction
>1. 각 이슈는 JSON 형식으로 출력해주세요.
>2. 각 이슈는 제목(title)과 설명(explain)으로 구성해주세요.
>3. 최신 데이터를 조사하여 100% 사실에 기반한 정보만 작성해주세요.  
>
>### 예시 형식
>```json
>{
>  "issues": [
>    {
>      "title": "AI 반도체 시장의 급격한 성장",
>      "explain": "고성능 AI 반도체 수요 증가로 인해 삼성전자, SK하이닉스 등 국내 기업들이 AI 전용 반도체 개발에 박차를 가하고 있으며, 정부도 AI 인프라 확충을 위한 정책을 추진하고 있습니다."
>    },
>    {
>      "title": "생성형 AI를 활용한 콘텐츠 혁신",
>      "explain": "AI가 텍스트, 이미지, 음악, 영상 등 다양한 콘텐츠를 자동 생성하면서, 광고, 마케팅, 교육, 엔터테인먼트 산업에서 활용도가 증가하고 있습니다."
>    },
>    {
>      "title": "AI 기반 자동화의 확산과 일자리 변화",
>      "explain": "기업들이 AI를 활용한 업무 자동화를 도입하면서 생산성과 효율성은 높아지고 있지만, 일부 직무의 대체 가능성으로 인해 새로운 직업군 형성에 대한 논의가 활발해지고 있습니다."
>    }
>  ]
>}
>```

>[!example] 블로그 작성 프롬프트
>```markdown
>## 🎯 Framing (역할 및 맥락 설정)
>당신은 10년 경력의 SEO 전문가이자 콘텐츠 전략가, 그리고 전문 작가입니다. 최적화된 블로그 콘텐츠를 작성하며, 검색 엔진 상위 노출과 독자 참여를 동시에 달성하는 것을 목표로 합니다.
>
>당신의 전문 분야:
>- SEO 키워드 최적화 및 의미론적 키워드 활용
>- 독자 중심의 스토리텔링
>- 데이터 기반 콘텐츠 전략
>
>## 📋 Request (요청 사항)
>주어진 키워드와 주제를 기반으로 SEO에 최적화된 블로그 콘텐츠를 작성해주세요.
>
>### 필수 요구사항
>1. 타겟 키워드를 포함하는 매력적인 제목 (H1)
>2. 콘텐츠를 요약하고 타겟 키워드를 포함하는 메타 설명 (150-160자 이내)
>3. 독자의 관심을 끌고 글의 맥락을 설정하는 도입부
>4. 주요 하위 주제를 다루는 메인 제목 3개 (H2s)
>5. 각 H3는 해당 H2s에 대한 최대한 상세 설명 문장, 글자 수는 제한 없음
>6. 각 H3에서 표, 목록, 또는 기타 시각적 요소를 포함하여 정보를 명확하게 전달
>7. 주요 포인트를 요약하고 행동 촉구(CTA)나 최종 생각을 제시하는 결론
>
>## 📚 Reference (참조 정보)
>
>### 작성 톤 & 스타일:
>- 어조: 권위 있지만 친근한 톤
>- 문체: 짧은 문장과 긴 문장 혼합 사용
>- 시점: 필요시 1인칭 시점 사용 가능
>- 감정 표현: 느낌표 사용하여 인간적인 느낌 유지
>
>### SEO 최적화 체크리스트:
>[ ] 타겟 키워드를 H1, H2에 자연스럽게 배치
>[ ] 의미론적 키워드를 본문에 분산 배치
>[ ] 내부/외부 링크 제안 (필요시)
>[ ] 이미지 alt 텍스트 제안
>
>### 콘텐츠 신뢰성 강화 요소:
>- 최신 데이터 및 통계 인용
>- 전문가 의견 또는 연구 결과 참조
>- 실제 사례 또는 케이스 스터디 포함
>
>## 📐 Format (출력 형식)
>아래 구조에 따라 콘텐츠를 작성해주세요: 
>
>1️⃣ 메타데이터
>H1: [타겟 키워드 포함 제목]
>메타 설명: [150-160자 요약]
>
>2️⃣ 도입부 (Introduction)
>- 독자의 관심을 끄는 오프닝
>- 흥미로운 통계, 질문, 또는 인용구로 시작
>- 글의 목적과 독자가 얻을 가치 명확히 제시
>
>3️⃣ 본문 (Main Content)
>H2: [첫 번째 주요 주제]
>- 개요 (불릿 포인트 형식)
>
>H3: [첫 번째 하위 주제]
>- 서술형 상세 설명 - 글자 수 제한 없음
>
>H3: [두 번째 하위 주제]
>- 서술형 상세 설명 - 글자 수 제한 없음
>
>[표 또는 목록 예시]
>| 항목 | 설명 |
>|------|------|
>| ... | ... |
>
>H2: [두 번째 주요 주제]
>- 위와 동일한 구조 반복
>
>H2: [세 번째 주요 주제]
>- 위와 동일한 구조 반복
>
>4️⃣ 결론 (Conclusion)
>- 주요 포인트 3가지 요약
>- 행동 촉구(CTA): "지금 바로 시작해보세요!", "댓글로 의견을 남겨주세요!"
>
>5️⃣ SEO 키워드 목록
>- 타겟 키워드: [주요 키워드]
>- 의미론적 키워드 (10개)
>
>✍️ 입력 정보

>[!danger] GPT-5.1 최적화 블로그 작성 프롬프트
>```markdown
><prompt>
><role>
>당신은 10년 경력의 SEO 전문가이자 콘텐츠 전략가, 그리고 전문 작가입니다. 목표는 주어진 키워드에 대해 검색엔진 상위 노출(SEO)과 독자 참여(읽기 쉬움·실용성)를 동시에 달성하는 고품질의 한국어 블로그 콘텐츠를 작성하는 것입니다.
></role>
>
><instructions>
>1. 먼저 아래 변수를 확인하고 이해하세요(이 변수들은 입력값이므로 여기에서 먼저 참조됩니다).
>{타겟 키워드: AI 자동화 n8n 사례}
>{SEO 필요한 추가 변수: 제목, 메타 설명, 도입부, 주요 주제(H2), 상세 설명(H3), 표 또는 목록, 결론, 키워드 목록 등}
>
>2. 사전확인(필수)
>   - 사용자가 추가 요구(길이, 톤, 대상 독자, 반응 유도 목표 등)를 명시하지 않았다면, <thinking_process>에서 적절한 사전질문을 제안하세요. 예: "목표 독자(개발자/마케터/기업담당자)는 누구인가요?", "원하시는 글 길이는 어느 정도인가요?".
>   - 사용자가 즉시 작성 요청을 하면, 기본 가이드를 적용(중간 길이, 일반 독자 대상, 전문·친절 톤).
>
>3. 리서치(재귀적·근거중심)
>   - 초안 작성 전 반드시 최신 데이터·전문가 의견·실제 사례를 확인하세요. 다음 함수들을 필요에 따라 호출하세요:
>     <function_call>cite_latest_data(query="...", limit=5)</function_call>
>     <function_call>cite_expert_opinions(topic="...", limit=5)</function_call>
>     <function_call>fetch_case_studies(topic="...", limit=5)</function_call>
>   - 각 호출 결과에서 핵심 인용문/수치/사례를 추출하고 출처(URL·저자·연도)를 기록하세요.
>   - 정보를 수집한 뒤, 결과에 모순이나 증거 부족이 있는지 점검하세요. 필요하면 추가 리서치(재귀 호출)를 수행해 근거를 보강하세요.

>- 인용구: 중요한 팁 
>- 표: 기능 비교

>

>[!question] 프롬프트 엔지니어링 vs 컨텍스트 엔지니어링
>
| 구분     | 프롬프트 엔지니어링       | 컨텍스트 엔지니어링                   |
| ------ | ---------------- | ---------------------------- |
| 초점 | 프롬프트(질문) 자체를 최적화 | 정보 환경과 시스템 맥락 설계             |
| 범위 | 단일 프롬프트 작성       | 외부 데이터, API, 도구 통합           |
| 목표 | 좋은 답변 얻기         | LLM의 성능을 극대화하는 환경 구축         |
| 기술 | Few-shot, CoT 등  | RAG, Function Calling, MCP 등 |
